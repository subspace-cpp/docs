Subspace

The Subspace C++ Library

The Subspace C++ Library provides a concept-centered abstraction on top of the bare-metal C++ standard library. It provides the tools to build stable applications quickly, and to make your application performant through explicitly leveraging compiler optimizations without accidentally tripping over Undefined Behaviour and miscompilation. Stop spending time debugging tricky heisenbugs and start relying on the compiler to produce the program you wrote.

Find Subspace on Github here: https://github.com/chromium/subspace

Type Aliases
  • Array

    A collection of objects of type T, with a fixed size N.

  • IntoRef
  • IterRefCounter

    Reference counting outstanding iterators (and view types since they need to be able to produce iterators) in order to catch iterator invalidation and prevent them from being used afterward. Mutating the collection should check that the count is empty. This is much like a RefCell in Rust, using runtime verfication that modification does not occur while there are outstanding references.

  • IteratorBase

    The base class for all Iterator types.

  • NoCopyMove
  • NonNull

    A pointer wrapper which holds a never-null pointer.

  • OkVoid

    A marker type for constructing Result with an Ok(void) value.

  • Option

    The Option type.

  • Result
  • Slice

    A dynamically-sized const view into a contiguous sequence of objects of type const T.

  • SliceMut

    A dynamically-sized mutable view into a contiguous sequence of objects of type T.

  • Tuple

    A Tuple is a finite sequence of one or more heterogeneous values.

  • Vec

    A resizeable contiguous buffer of type T.

  • f64

Function Aliases
  • clone

    Clones the input either by copying or cloning. Returns a new object of type T.

  • clone_into

    Performs copy-assignment from source.

  • ensure_use
  • replace

    Replace the dest with src, and return the old value in dest.

Variable Aliases
  • unsafe_fn

    The global UnsafeFnMarker which can be passed to unsafe functions. See the UnsafeFnMarker type for an explanation.

Concept Aliases
  • Clone

    A Clone type can make a new copy of itself.

  • Copy

    A Copy type can be copied to construct a new object and can be assigned to by copy.

  • Default

    Default types are able to be constructed with a default value.

  • From

    A concept that indicates ToType can be constructed from a FromType, via ToType::from(FromType).

  • Into

    A concept that declares FromType can be converted to ToType.

  • TryFrom

    A concept that indicates ToType can be (sometimes) constructed from a FromType, via ToType::try_from(FromType).

  • relocate_by_memcpy

    Tests if a variable of type T can be relocated with copy.
